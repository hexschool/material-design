# 寫作

文字內容應該要不論文化或語言，在任何地方的任何人都能夠理解。

# 原則

使用者介面文字能讓介面更易讀並建立信任感，文字必須明白易懂、準確並且簡明扼要。

相關文章：共通寫作 (opens new window)

# 要精準

為了促進導覽與探索，撰寫使用者介面文字時要簡短，在同一時間內專注在限定幾個概念，寫成可快速一覽的區段。

轉帳給任何在美國有電子郵件的人，快速、簡單而且免費。

該這樣做

精準地表達資訊及行動。

用電子郵件在美國轉出（以及收受）錢給朋友及家人。這是個低風險的兩步驟方法，而且不會對接收者收取任何手續費。

別這樣做

當介紹一個關鍵的行動或概念時，不要羅列好幾個複雜的概念。

閱讀您的手機內建文件

該這樣做

撰寫指引時專注在少量的關鍵概念。

更多用法說明，請查閱您的手機內建文件

別這樣做

別將指引嵌入在冗長的敘述中。

# 簡單直接地寫

使用簡單且直接的語句讓文章更好懂。

儲存變更？

該這樣做

保持介面文字簡短。

請問您想要儲存您的變更嗎？

別這樣做

傳達一個行動時，別把指引寫得長於必要的長度。

註冊以投票

該這樣做

介面文字應專注於目前的任務。

在您可以投票前必須先註冊

別這樣做

告訴使用者如何採取行動，而非告訴他們應該或者必須做這件事。

# 清楚指稱使用者

用第二人稱（你（您）、你（您）的）或者第一人稱（我、我的）指稱使用者，根據當下情況選擇最適合、最清楚的人稱。 各人稱根據以下狀況建議使用：

  • 第二人稱，「你（您）」或者「你們」，這種對話風格幾乎適用於所有情況，如同使用者介面直接與使用者對話。
  • 第一人稱，「我」或者「我們」，在部分情況下，這種人稱形式用於強調使用者對於內容或者行為的所有權。

使用第一人稱強調使用者的行為所有權

該這樣做

直接以第二人稱對使用者說話

我同意遵循服務條款

該這樣做

不需解鎖螢幕就可以快速開啟您的相機

您的空間

該這樣做

直接以第二人稱對使用者說話

我的帳戶

該這樣做

使用第一人稱強調內容所有權。

# 避免混用第一人稱及第二人稱

為避免混淆使用者，避免在同一段落中同時使用「我」、「我的」、「你（您）」、「你（您）的」。

在我的帳戶中改變您的偏好設定

別這樣做

別在同一段落中同時使用第一人稱及第二人稱指稱使用者。

# 傳達必要的細節

只傳達必要的細節，讓使用者能專注於他們自己的任務。有時最有效率的使用者介面沒有包含任何文字。

登入中⋯⋯

您的電話正在聯繫我們，這可能會花費數至 5 分鐘的時間。

該這樣做

僅傳達理解現在狀態及行動的必要細節

登入中⋯⋯

您的電話需要與我們的伺服器連結以登入您的帳戶，這可能會花費數至 5 分鐘的時間。

別這樣做

避免提供使用者非必要的細節，例如該行動或程序是如何執行的。

# 使用每個人都能理解的單詞

使用對任何閱讀能力都清楚的、容易理解的常見字眼。

打開位置紀錄

該這樣做

使用常見的字眼

授權位置紀錄

別這樣做

# 專業術語及功能名稱

避免使用專業術語或者為使用者介面發明的功能名稱。

準備影片中⋯⋯

該這樣做

使用常見的字眼

緩衝中⋯⋯

別這樣做

避免使用技術性行話，除非對於理解內文是不可或缺的

這個指令不支援您的裝置

該這樣做

使用使用者介面及裝置都常用的敘述方法

這個指令僅支援雙核心裝置

別這樣做

避免使用技術性行話，除非對於理解內文是不可或缺的

# 用現在式書寫

用現在式描述產品行為，避免使用未來式來描述產品的通常行為。

當你需要用過去式或現在式書寫時，用簡單式動詞。

訊息已發送

該這樣做

用現在式書寫

訊息已經被發送

別這樣做

別用不同的方式書寫現在式，例如現在完成式。

# 使用阿拉伯數字

使用阿拉伯數字（「1、2、3」 而非「一、二、三（壹、貳、參）」）除非文案將數字混合使用，例如「輸入兩個 3」。

您有 3 封訊息

該這樣做

使用阿拉伯數字。

輸入兩個 3

別這樣做

當用兩種不同的方法使用數字時，一個使用數字表達、另一個使用文字。

您有三（three）封訊息

該這樣做

別把數字單字拼出來。

# 省略非必要的標點符號

為了讓使用者能一眼瀏覽內文，避免在非必要之處使用標點符號。

# 句號

在介面元素中，避免在單獨的句子中使用句號。

  • 標籤、標註
  • 懸停文字
  • 子彈清單
  • 對話體裁內文 (opens new window)

使用句號的時機：

  • 多個句子
  • 任何後面跟隨連結的句子（連結本身不應以全長呈現）

# 較長段落中的句號

較長或者複雜的句子，在合適情況下可使用句號。例如，若在你應用程式的其他動線、流程中皆沒有使用句號，只將他們呈現在少數地方會顯得不一致。

還原大量修改？

如果您還原此大量修改，所有您做的變更將會返回至原始狀態

該這樣做

主體中的單一句子省略句號

還原大量修改？

所有您做的變更將會返回至原始狀態。

別這樣做

如果只有單一句子，在文字主體後不要加入句號

# 冒號

標籤後省略冒號

分享

該這樣做

分享：

別這樣做

# 內容結構

# 以目的開頭

若文字段落描述一需完成的特定目標或動作，以該目標作為開頭。

若要從相簿中移除一張照片，將它拖曳至垃圾桶

該這樣做

以目的開始（「移除照片」）並以使用者行動結尾（「拖曳至垃圾桶」）

將照片拖曳到垃圾桶以從此相簿刪除它

別這樣做

不要在目標（“將其從此相冊中刪除”）之前聲明使用者採取的操作（“拖動照片”）。

# 在需要時揭示細節

在需要時循序地揭示資訊。

在使用者開始互動時，不需要將所有的細節一次敘述。當使用者繼續探索並需要更多資訊時，再揭示更多功能細節。

移除已下載的書籍？

該這樣做

你確定你想要移除此已下載書籍？你將於連接網路時才能存取。

別這樣做

# 使用一致的字彙

在介面功能中使用一致的字彙。

移除照片

在頁面中移除照片？

該這樣做

在介面中用一致的語言描述同一動作

移除照片

刪除照片？

別這樣做

在介面中不要使用不同的動詞、語句描述同一動作

# 透過標誌引用或者控制元件

標誌本身會表明控件或者元件的功用。它們會出現在附近或者控件本身，例如在按鈕或者開關上的文字。若要引用介面控件，使用他們自己的標誌或文字。（不要提到該元件自身的種類）

點擊繼續

該這樣做

只敘述介面標誌

點擊繼續按鈕

別這樣做

不要在元件本身上提到標誌與自身
